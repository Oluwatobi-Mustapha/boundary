security: fix critical vulnerabilities in access provisioning

This commit addresses 4 high-severity security issues identified in the
security audit:

## Fixed Issues

### H-1: Race Condition in Access Provisioning (CRITICAL)
- **Impact**: Zombie access grants if DynamoDB write fails after AWS provisioning
- **Fix**: Reordered operations to save state BEFORE provisioning
- **Flow**: PENDING → Provision → ACTIVE (atomic with rollback capability)
- **Files**: src/main.py

### H-2: Missing Duration Input Validation
- **Impact**: DoS via negative/extreme duration values, TTL overflow
- **Fix**: Added validation (0 < duration <= 720 hours)
- **Files**: src/validators.py, src/main.py

### H-3: Unvalidated Account ID Input
- **Impact**: Enumeration attacks, injection attempts, system crashes
- **Fix**: Added 12-digit format validation with regex
- **Files**: src/validators.py, src/main.py

### H-4: Audit Log Credential Leakage
- **Impact**: Sensitive AWS account IDs exposed if logs committed to Git
- **Fix**: Added audit_logs/ to .gitignore
- **Files**: .gitignore

### M-5: Environment Variable Format Validation (Bonus)
- **Impact**: Misconfiguration with invalid OU IDs
- **Fix**: Added format validation for OU_ID and Root ID patterns
- **Files**: src/core/engine.py

## Testing
- Added comprehensive unit tests for all validators
- Tests cover edge cases: negative values, SQL injection, format violations
- Run: `pytest tests/test_validators.py -v`

## Breaking Changes
None. All changes are backward compatible.

## Deployment Notes
- Ensure environment variables are set before deployment
- Test with invalid inputs to verify fail-fast behavior
- Monitor CloudWatch logs for validation errors
